# Advanced Usage

In this section, we will cover some advanced usage scenarios that you might encounter while working with the API client.

## Custom Headers

You can set custom headers for all requests by providing a `headers` object in the configuration object.

```ts
const config = {
  baseURL: 'https://jsonplaceholder.typicode.com',
  headers: {
    'Content-Type': 'application/json',
    Authorization
  },
};

const Api = new ApiClient(config);
```

### Overriding Headers

You can also override the default headers for a specific request by providing a `headers` object in the request options.

```ts
Api.get('/posts', {
  headers: {
    'Accept': 'application/json',
  },
}).then((response) => {
    console.log(response.data);
});
```

## Parameter Serialization

You can serialize request parameters using an object 

```ts
const forecast = await Api.get(
    'forecast',
    {
        latitude: 51.5074,
        longitude: -0.1278,
        current_weather: true
    },
    {
        config: {
            baseUrl: 'https://api.open-meteo.com/v1'
        }
    }
);
```

### With Url Parameters

You can also provide URL parameters in the request options.

> This one below is the same as the one above, but with URL parameters.

```ts
const forecast = await Api.get(
    'forecast?latitude=51.5074&longitude=-0.1278&current_weather=true',
    {
        config: {
            baseUrl: 'https://api.open-meteo.com/v1'
        }
    }
);
```


## Parameter Swapping

You can swap parameters in the URL by providing an object with the parameter names and values in the request options.

**Params First:**
```ts
const forecast = await Api.get(
    'forecast',
    {
        latitude: 51.5074,
        longitude: -0.1278,
        current_weather: true
    },
    {
        config: {
            baseUrl: 'https://api.open-meteo.com/v1'
        }
    }
);
```

> This one below is the same as the one above, but with the parameters swapped.

**Params Last:**
```ts
const forecast = await Api.get(
    'forecast',
    {
        config: {
            baseUrl: 'https://api.open-meteo.com/v1'
        }
    },
    {
        latitude: 51.5074,
        longitude: -0.1278,
        current_weather: true
    }
);
```

## Parallel Requests

You can make parallel requests using the `Promise.all` method.

```ts
// Execute multiple requests in parallel
Api.all({
  users: Api.get('/users'),
  posts: Api.get('/posts'),
  comments: Api.get('/comments')
})
  .then(results => {
    console.log('Users:', results.users);
    console.log('Posts:', results.posts);
    console.log('Comments:', results.comments);
  });
```

### Parallel Requests with Async/Await
```ts
const { comments, forecast } = await Api.all({
    comments: Api.get('/comments?postId=1', {
        config: {
            dataKey: ''
        }
    }),
    forecast: Api.get(
        'forecast',
        {
            config: {
                baseUrl: 'https://api.open-meteo.com/v1'
            }
        },
        {
            latitude: 51.5074,
            longitude: -0.1278,
            current_weather: true
        }
    )
});
// Access results using the property names
console.log("comments", comments);  // comments data
console.log("forecast", forecast);  // forecast data
```

## Token Management

The ApiClient provides built-in support for managing authentication tokens. This is particularly useful for OAuth 2.0 flows where you need to handle both access tokens and refresh tokens.

### Setting Tokens

```ts
const api = new ApiClient({ baseUrl: 'https://api.example.com' });

// Set individual tokens
api.setAccessToken('eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...');
api.setRefreshToken('dGVzdC1yZWZyZXNoLXRva2Vu');

// Or set both tokens at once
api.setTokens(
  'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',  // access_token
  'dGVzdC1yZWZyZXNoLXRva2Vu'                    // refresh_token
);
```

### Getting Tokens

```ts
// Get individual tokens
const accessToken = api.getAccessToken();
const refreshToken = api.getRefreshToken();

// Check if tokens exist
if (accessToken) {
  console.log('User is authenticated');
}
```

### Clearing Tokens

```ts
// Clear all tokens (useful for logout)
api.clearTokens();
```

### Using Tokens with Interceptors

Combine token management with interceptors for automatic authentication:

```ts
// Automatically add access token to all requests
api.interceptors.request.use(async ({ config, options }) => {
  const token = api.getAccessToken();
  
  if (token) {
    options.headers = {
      ...options.headers,
      'Authorization': `Bearer ${token}`
    };
  }
  
  return { config, options };
});

// Handle token refresh on 401 responses
api.interceptors.response.use(
  ({ data, response }) => ({ data, response }),
  async (error) => {
    if (error.status === 401 && api.getRefreshToken()) {
      // Refresh token logic here
      const newTokens = await refreshTokens(api.getRefreshToken());
      api.setTokens(newTokens.access_token, newTokens.refresh_token);
      
      // Retry original request...
    }
    return Promise.reject(error);
  }
);
```

## File Uploads

You can upload files by providing a `FormData` object as the request data.

```ts
// File upload with a single file
const fileInput = document.querySelector('#fileInput');
api.upload('/upload', 
  { profileImage: fileInput.files[0] },
  { userId: 123 }
)
  .then(data => console.log('Upload successful:', data));
```

### Multiple File Uploads
```ts
// Upload multiple files
const multipleFiles = document.querySelector('#multipleFiles');
api.upload('/upload-gallery',
  { images: Array.from(multipleFiles.files) },
  { albumId: 456 }
)
  .then(data => console.log('Gallery upload successful:', data));
```
